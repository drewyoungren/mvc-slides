Ephemeral WebRTC Chat Prototype

One static file. No application backend. STUN only. Manual out-of-band signaling. This first cut supports one presenter and one attendee connection at a time. It is intentionally dumb so we can verify the handshake.

How to test

  1. Open this file in two separate browsers or devices. Use Presenter mode on one side and Attendee mode on the other.
  2. Presenter clicks Create invite and copies the blob into the attendee's Presenter invite blob field.
  3. Attendee clicks Create answer and copies the returned blob back into the presenter's Attendee answer blob field.
  4. Wait a few seconds for ICE to finish. If the data channel opens, chat should work.

This prototype uses the public STUN server stun:stun.l.google.com:19302. There is no TURN fallback, so some networks will fail.
